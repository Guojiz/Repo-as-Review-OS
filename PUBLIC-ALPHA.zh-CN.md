# 早期公开版本

GitLearnOS 是可试用的早期个人学习控制层。它可以交给一个具备工具能力的主 Agent 运行；整理、出题、自动化与模板仍会根据真实使用反馈调整。

仓库继续保留历史名称 `Repo-as-Review-OS`。

## 已可测试

- ChatGPT Work 与仓库 Agent 的单一主 Agent 直接路径；
- 老师、课堂、纸笔、题库、AI等跨渠道输入；
- 自动整理、关联、去重和外部反馈同步；
- 个性化诊断题、变式题、迁移题、复习题与老师问题包；
- `safe-auto` / `preview` / `manual` 用户政策；
- 即时自动写回与接手检查；
- 来源、学习画像、模型、知识缺口、问题包、复习和活动模板；
- 处理状态与掌握证据分离；
- 基于证据的 0–3 评分与可复现复习间隔；
- 从问老师到反馈同步再到自动出题的中文 demo。

## 这样试用

```text
把 https://github.com/Guojiz/Repo-as-Review-OS 作为 GitLearnOS 模板。
目标仓库：<私有仓库链接>
学习目标或当前输入：<内容>

读 START-HERE.zh-CN.md 与 AGENTS.zh-CN.md，采用 safe-auto。
自动整理，需要时针对性出题，并实际写回安全变化。不要默认进入
AI 教学会话。汇报改动、证据、实际自动化、下一步和撤销能力。
```

## 仍在实验

- 不同学科的题型与难度适配；
- 自动去重和状态关联在长期使用中的准确性；
- 外部反馈后默认验证的最佳强度；
- 0–3 间隔是否适合所有目标；
- 真实定时调度器的可选适配；
- 更长时间的跨平台可移植性。

## 对标边界

港大 DeepTutor 是能力对标。GitLearnOS 不声称功能或 Runtime 对等；它强调跨渠道、用户所有权和低部署成本。原始 `zhongkao` 仓库是实践基线。

## 安全

真实状态默认使用私有目标仓库。版权原件、私人截图、老师原文件、凭据和敏感身份不能进入公开模板。自动化必须可见、低风险并尊重用户政策。
