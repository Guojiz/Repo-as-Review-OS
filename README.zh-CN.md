# 仓库即复习操作系统

[English](README.md)

这是一个基于 GitHub 和 Markdown 的可部署复习系统模板。

核心想法很简单：把一个仓库当成复习的操作系统。笔记、错题、可复用方法、写作草稿、来源检查和接手规则，不应该散落在聊天记录和截图里，而应该进入一个清晰的结构。

## 目标

- 保持原始材料可追踪。
- 把错题转化为可复用模型。
- 区分原文、摘要、索引和待补项。
- 让后续工具在编辑前先有地图。
- 只公开清理后的方法层内容。

## 建议目录

```text
Repo-as-Review-OS/
├── README.md
├── README.zh-CN.md
├── AGENTS.md
├── AGENTS.zh-CN.md
├── LICENSE
├── docs/
├── agents/
├── templates/
└── website/
```

## 主要层级

- `docs/`：架构、部署、隐私与来源规则。
- `agents/`：AI 或后续编辑者的接手与维护规则。
- `templates/`：错题卡、写作素材卡、每日复盘和控制台模板。
- `website/`：公开网页文案和一个可切换语言的小页面。

## 最小工作流

1. 加入原始材料、错题、草稿或反馈。
2. 标记状态：原文、摘要、索引、模型或待补。
3. 把可迁移内容转化为模板。
4. 编辑前先读接手规则。
5. 只部署清理后的公开内容。

## 给 AI Agent

修改文件前先阅读 `AGENTS.zh-CN.md`。如果使用英文环境，也可以读 `AGENTS.md`。
