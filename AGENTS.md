# AGENTS.md / AI 接手入口

This file is for ChatGPT, Codex, and other AI agents. Keep it short and follow it before editing this repository.  
本文件给 ChatGPT、Codex 和其他 AI Agent 使用。请保持简短，并在编辑仓库前先遵守这里的规则。

## Mission / 任务

Repo as Review OS is a reusable review-system template.  
Repo as Review OS 是一个可复用的复习系统模板。

Your job is to help users turn scattered study material into a structured repository with sources, models, templates, dashboards, and handoff rules.  
你的任务是帮助用户把分散的复习材料整理成结构化仓库：包含来源、模型、模板、控制台和接手规则。

## Read first / 先读

1. `README.md`
2. `docs/architecture.md`
3. `docs/privacy-and-sourcing.md`
4. `agents/handoff-protocol.md`
5. Relevant files in `templates/`

## Do / 应该做

- Keep original sources traceable. / 保持原始来源可追踪。
- Mark incomplete files as `todo`. / 不完整文件标记为 `todo`。
- Use bilingual headings when adding public docs. / 添加公开文档时使用中英双语标题。
- Prefer small, reviewable changes. / 优先做小而可审查的改动。
- Explain changed files after editing. / 编辑后说明改了哪些文件。

## Do not / 不要做

- Do not invent missing study context. / 不要编造缺失的学习上下文。
- Do not treat a summary as a source. / 不要把摘要当作来源。
- Do not publish private examples. / 不要公开私人样例。
- Do not remove user data unless explicitly asked. / 未明确要求时不要删除用户资料。

## Output format / 输出格式

After work, report:  
完成后报告：

1. Files changed / 修改文件；
2. What changed / 改动内容；
3. What remains / 剩余事项。
