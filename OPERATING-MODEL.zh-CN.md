# 运行模型

Repo as Review OS 是一个基于 GitHub 的目标驱动型 AI 学习操作系统。

它不只是复习模板，而是一种让一个主 AI 工具帮助用户学习、记忆、重组，并从旧材料中生成新理解的方法。

## 核心论点

用 GitHub 作为可写入的学习时间线。

用一个主 AI 工具作为维护者。

用目标决定什么重要。

用文件和目录表达状态。

用间隔复习防止遗忘。

用自动化整理仓库和生成题单。

## AI 应该按什么顺序执行

当 AI 工具拿到这个仓库时，应该按这个顺序：

1. 判断自己的权限等级。
2. 读取用户目标。
3. 检查仓库结构。
4. 找到活跃来源、模型和薄弱点。
5. 判断什么要拆解、合并、重写、复习、生成或归档。
6. 更新文件。
7. 必要时安排下次复习。
8. 汇报每一次改动。

## 仓库是用来做什么的

仓库不应该收集所有东西。

它应该转化材料：

```text
原始材料
→ 来源记录
→ 提取项目
→ 可复用模型
→ 绑定目标的复习任务
→ 生成题单
→ 更新学习状态
```

## GitHub 提供什么

GitHub 给 AI 提供：

- 文件树；
- 写回能力；
- 历史记录；
- 时间线；
- 可追踪改动；
- 共享操作层；
- 自动化输出位置。

这就是为什么普通 AI 项目本身还不够。

## 记忆提供什么

记忆或项目规则应该保存用户的稳定偏好：

- GitHub 是主要操作层；
- 本地文件是受保护来源；
- 默认使用一个主 AI 工作流；
- 中文和英文分文件；
- 每次改动都要汇报。

## 目标提供什么

目标防止仓库变成文件堆。

每个文件都应该回答：

```text
它服务于哪个目标？
```

如果一个文件不服务当前或未来目标，就应该归档或降低优先级。

## 复习提供什么

复习不是最终目标。复习是学习的地基。

系统回到旧材料，是为了让用户从旧材料中学到新东西。

## 自动化提供什么

自动化主要做两件事：

1. 整理仓库；
2. 根据目标、薄弱点、历史和间隔复习生成题单。

## 最终形态

最简单的形态是：

```text
一个 AI App
+ 一个 GitHub 仓库
+ 一个连接器
+ 记忆或项目规则
```

这已经足够从手机或电脑运行一个个人学习操作系统。
