# 运行模型

GitLearnOS 是轻量教学套件，它把当前智能执行与长期学习状态分开。

## 三层结构

```text
当前 AI Runtime
→ 教学、检索、提问、观察与操作

GitLearnOS Skills 与规则
→ 保证学习闭环、证据标准与写回政策

目标 GitHub 仓库
→ 保存学习者拥有、可检查的状态与历史
```

## 核心闭环

```text
目标
→ 来源或已有模型
→ 学习者尝试
→ 诊断缺口
→ 自适应支持
→ 新检验
→ 证据评分
→ 选择性写回
→ 下次复习或行动
```

## 状态模型

- `dashboard.md`：当前链接视图与下一步；
- `learner-profile.md`：有证据支撑的长期学习者状态；
- `goals/`：结果与成功标准；
- `sources/`：来源、访问能力与实际检查材料；
- `models/`：可复用理解；
- `knowledge-gaps/`：可观察薄弱点与解决标准；
- `reviews/`：题目、回答、分数与下次日期；
- `sessions/`：只有会话改变状态时保存简短重要证据。

## 决策规则

创建或修改文件前先问：

```text
是否服务于活跃目标？
是否由可用证据支撑？
未来是否仍有学习价值？
是否已经有唯一文件拥有这项状态？
```

否则留在当前对话中，或不创建文件。

## 对标边界

DeepTutor 提供能力方向：来源支撑、持续学习者状态、个性化任务和闭环教学。GitLearnOS 将这些方向压缩成由外部 Agent 执行的可移植文件与 Skills。

它有意不提供完整应用、服务端、多 Agent Runtime、RAG 引擎或数据库。

## 最小可用系统

```text
一个有能力的 AI Agent
+ 一个私有目标仓库
+ 一个活跃目标
+ 一份统一 Agent 规则
= 可以开始
```

其他组件只有在能改善学习时才加入。
