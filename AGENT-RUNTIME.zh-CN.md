# AI 运行指南

GitLearnOS 不提供自己的模型 Runtime。它把一套轻量教学协议挂载到学习者已经使用的 AI 环境中。

## 能力层级

### A 级——具备工具能力的工作区

当前环境能读取模板、检查授权来源并写入目标仓库。已正确连接的 ChatGPT Work 会话就是一种例子。

直接执行：

```text
验证权限 → 检查目标状态 → 完成学习任务 → 写回 → 验证
```

### B 级——可操作仓库的工作区

当前环境在本地或云端 checkout 中工作。遵守仓库指令，保留无关改动，使用同样的学习会话协议，并且只在获授权的流程中提交。

### C 级——只读或原生聊天

当前环境可以教学，也可能读取选定上下文，但不能写入目标仓库。输出简短待写回区块，绝不能声称 GitHub 已改变。

### D 级——单上下文回退

用户提供活跃目标、相关画像摘录和一份任务相关状态文件。完成一个学习闭环，不得声称拥有长期记忆或文件权限。

## 最小读取集合

先读 `START-HERE.zh-CN.md`、`AGENTS.zh-CN.md`，然后只读活跃目标文件。不要预加载所有平台指南、模板、Demo 或 Skills。

## 当前 Runtime 的责任

- 教学并让学习者先尝试；
- 通过现有工具访问获授权来源；
- 诊断可观察知识缺口；
- 调整讲解与练习；
- 用新检验验证；
- 可以时写入长期证据；
- 诚实说明能力限制。

## 状态责任

```text
目标仓库 → 持续变化学习状态的唯一真相
learner-profile.md → 可检查学习者状态
Skills / 规则 → Agent 行为
原生记忆 → 可选的稳定偏好缓存
连接或本地原件 → 获授权的来源界面
当前对话 → 临时工作上下文
```

## 权限规则

已获授权时，可以完成学习请求所需的普通目标状态写入。删除、大范围覆盖、公开状态变化、发布私人内容、接触密钥或更换许可证前必须询问。

## 单一主 Agent 默认

默认使用一个主 AI 工作流。子 Agent、本地工具、RAG、可视化或自动化都是可选能力，不是要求。共享的唯一状态必须保留在目标仓库。
