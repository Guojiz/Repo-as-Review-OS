# 贡献指南

感谢你考虑为 GitLearnOS 做贡献。

这个项目目前是早期公开版本。贡献应该帮助它更容易理解、更安全、更适合真实学习工作流。

仓库名仍然是 `Repo-as-Review-OS`，用于保持链接连续。

## 这个项目是什么

GitLearnOS 是一个基于 GitHub 的目标驱动型 AI 学习操作系统。

它不是用来上传私人学习资料的地方。这个仓库应该保存公开模板、方法和清理后的示例。

## 适合贡献什么

你可以贡献：

- 更清楚的解释；
- 更好的目录结构；
- 更安全的 AI 运行规则；
- 新模板；
- 新 demo；
- 自动化提示词；
- issue 模板；
- 翻译；
- 非考试场景示例；
- 使用 AI 工具试跑后的反馈。

## 不要贡献什么

请不要贡献：

- 私人学习记录；
- 老师文件；
- 有版权的教材页面；
- 私人截图；
- 真实考试记录；
- token、cookie、账号信息或其他密钥；
- 任何不能公开分享的内容。

## 推荐贡献流程

1. 先读 `README.zh-CN.md`。
2. 再读 `START-HERE.zh-CN.md`。
3. 再读 `PUBLIC-ALPHA.zh-CN.md`。
4. 看 `examples/zh-CN/demo-zhongkao-lite/`。
5. 大改动前先开 issue。
6. 保持改动小而清楚。
7. 说明你的改动解决了什么问题。

## demo 贡献

一个好的 demo 应该展示完整学习循环：

```text
目标
→ 来源或来源记录
→ 模型
→ 知识缺口
→ 题单
→ 复习安排
→ 控制台
```

只能使用虚构或清理后的材料。

## 模板贡献

一个好的模板应该包含：

- 目标链接；
- 来源状态；
- 知识缺口链接；
- 复习字段；
- 清楚的 AI 操作说明；
- 状态标签。

## AI 生成内容

可以使用 AI 起草贡献，但提交前应该由人类检查。

如果贡献使用了 AI 辅助，请说明。

## PR 检查清单

提交 PR 前请检查：

- 没有私人或版权材料；
- 长文档不要中英逐行混排；
- 改动符合运行模型；
- 新文件命名清楚、位置合理；
- demo 是虚构或清理后的；
- 必要时更新 README 或索引。

## 贡献态度

这个项目应该保持实用。

仓库服务学习，不要变成漂亮但不好用的文档花园。
