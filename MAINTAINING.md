# Maintaining

Use this checklist when reviewing changes or planning updates.

## Review priorities

- Keep the system useful for real learning workflows.
- Keep templates small enough for contributors to understand.
- Prefer examples that show a full loop: goal, source record, model, practice, review, dashboard.
- Keep private learning records out of the public repo.
- Make AI-agent instructions explicit and reviewable.

## Before accepting a pull request

- Check that no private records, copyrighted textbook pages, teacher files, tokens, or account data are included.
- Confirm demo content is fictional, cleaned, or public.
- Confirm the change supports the operating model.
- Ask for a smaller pull request if the change mixes unrelated concepts.

## When adding templates

A useful template should include:

- goal link;
- source status;
- review fields;
- clear AI instructions;
- status labels;
- a small example when helpful.

## Release notes

For notable changes, document:

- what changed;
- who benefits;
- migration notes;
- privacy or safety implications;
- any follow-up work.
