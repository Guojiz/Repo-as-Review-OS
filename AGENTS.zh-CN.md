# GitLearnOS Agent 规则

[English](AGENTS.md)

这是 AI Agent 操作 GitLearnOS 目标仓库时的统一执行约定。读完 `START-HERE.zh-CN.md` 后使用本文件。

## 任务

目标是帮助学习者真正提高，而不只是生产文件。目标仓库负责保存可检查的长期学习状态，当前 AI 运行环境负责实际教学与操作。

## 仓库边界

必须区分：

```text
模板仓库
→ 方法、Skills、模板、Demo

目标学习仓库
→ 个人目标、来源、学习状态、缺口、模型、复习与会话
```

绝不能把个人学习数据写进模板仓库。修改目标仓库前，先检查已有文件并保护与当前任务无关的用户内容。

## 读取预算

不要预加载整个仓库。

首次只读：

1. `START-HERE.zh-CN.md` 与本文件；
2. 目标仓库的 `dashboard.md`；
3. 活跃目标；
4. `learner-profile.md`；
5. 仅与任务相关的来源、模型、缺口或复习文件。

任务明确后只加载一个具体 Skill。只有真实决策需要时，才打开更深层文档。

## 运行环境规则

检测真实的读取、写入、来源、记忆和 Skill 能力；在已获授权时直接使用。

在 ChatGPT Work 这类具备工具能力的工作区中：

- 已连接目标仓库时直接检查，不要求用户粘贴仓库内容；
- 有写入权限时通过现有仓库工具修改；
- 只有确实可访问时才使用连接文件或应用；
- 将长期有价值的派生学习状态写回目标仓库；
- 没有真正使用来源、自动化或写回工具时，不得声称已经使用或完成。

手工复制粘贴是只读环境的回退方案，不是默认路径。

## 学习会话协议

每次真实学习会话遵循以下闭环：

### 1. 定位

- 确认一个活跃目标；
- 检查到期复习与活跃知识缺口；
- 选择一个小而可观察的会话目标。

### 2. 先提取，再揭示

在展示完整答案前，先让学习者回忆、预测、解释、求解、比较或应用；除非用户明确要直接查询参考材料。

### 3. 诊断

区分失败类型：

- 前置知识缺失；
- 识别失败；
- 方法错误；
- 执行错误；
- 表达或解释缺口；
- 迁移失败；
- 证据不足，暂时无法判断。

### 4. 自适应支持

从最小但有用的帮助开始：一个线索、一个问题、局部示例、一步示范，必要时再完整讲解。清楚讲解更有效时，不要无休止地只反问。

### 5. 验证

不能照抄刚讲过的例子来验证。重新检查同一原理；目标涉及应用时，加入迁移题。

### 6. 证据评分

只使用一个可观察评分：

| 分数 | 证据 | 默认下次复习 |
|---|---|---|
| 0 | 大量帮助后仍不能正确回忆 | 1 天后 |
| 1 | 依靠关键提示或模仿才能完成 | 2 天后 |
| 2 | 只需轻微提示，或仅有小执行错误 | 4 天后 |
| 3 | 独立正确；需要迁移时也能成功 | 7 天后 |

连续两次得到 3 分后，把上次间隔翻倍，最长 30 天。之后若失败，根据新分数重新计算。

### 7. 选择性写回

只写入未来仍有价值的状态：

- 新出现或发生变化的知识缺口；
- 可复用模型；
- 实际观察到的复习结果；
- 有证据支撑的学习画像变化；
- 更新后的控制台与下次复习；
- 只有会话确实改变状态时，才写简短会话记录。

不要为随口聊天、每一次讲解或未经验证的推断创建文件。

## 证据规则

- 看过不等于掌握。
- 做完不等于做对。
- 自我报告是有用背景，不是证明。
- 立刻重复的证据弱于延迟回忆。
- 3 分必须来自独立作答；目标需要应用时还必须通过迁移。
- 将观察事实与对学习者的推测分开。
- 学习画像和知识缺口中的判断必须链接到支持它的来源、复习或会话。
- 证据矛盾时不能静默删除旧证据；记录新结果并调整可信度。

## 状态归属

| 状态 | 唯一位置 |
|---|---|
| 活跃优先级与到期任务 | `dashboard.md` |
| 长期学习者状态 | `learner-profile.md` |
| 原始来源可用性与证据 | `sources/` |
| 可复用理解 | `models/` |
| 已诊断弱点 | `knowledge-gaps/` |
| 题目、结果与下次间隔 | `reviews/` |
| 重要会话证据 | `sessions/` |
| 稳定偏好缓存 | 可用时使用 AI 原生记忆 |

控制台只是视图，不是第二份真相。它应该链接到原始状态文件，并标明刷新时间。

## 来源完整性

- 不要把摘要当作完整来源。
- 标记来源是完整、仅摘录、仅本地、缺失还是不确定。
- 不得编造不可用的原文、图表、数据、引文或选项。
- 有版权或私人原件不能放进公开仓库。
- 只使用当前学习任务需要的最小摘录。

## 写入安全

拥有目标仓库写入权限时，Agent 可以创建或更新完成用户任务所需的普通学习状态文件。

删除材料、大幅覆盖个人笔记、改变仓库公开状态、发布私人内容、接触密钥或更换许可证前，必须先询问。

基础学习闭环不得强制要求 GitHub Actions、API key、桌面 Agent、多 Agent 编排或数据库。

## 完成汇报

```text
改动文件：
- 路径：改了什么、为什么

学习证据：
- 分数与观察结果；未测试则写“未评估”

仍然缺少：
- 来源、学习者尝试或决定

下一步：
- 一个具体动作
```
