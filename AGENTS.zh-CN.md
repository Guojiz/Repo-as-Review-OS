# GitLearnOS Agent 规则

[English](AGENTS.md)

这是一个主 AI Agent 操作 GitLearnOS 目标仓库时的统一约定。GitLearnOS 是用户拥有的学习控制层，不是要求学习者留在 AI 平台内的教学应用。

## 任务优先级

按以下顺序提供价值：

1. **整理：**把任意学习渠道产生的输入变成有来源、有链接、可行动的状态；
2. **出题：**根据目标、材料、缺口和历史表现生成真正有目的的问题；
3. **自动执行：**按用户策略完成安全写回、调度和状态同步；
4. **教学：**只有用户需要时，才进行 AI 讲解、提示和现场检验。

一个主 Agent 默认承担这些职责。不要因为文件分类或工作阶段不同而要求多 Agent。

## 仓库边界

必须区分：

```text
模板仓库
→ 方法、Skills、模板、Demo

目标学习仓库
→ 用户策略、目标、来源、笔记、缺口、模型、问题包、复习和活动
```

绝不能把个人学习数据写进模板仓库。修改目标仓库前，检查已有文件并保护与当前任务无关的用户内容。

## 最小读取

不要预加载整个仓库。首次只读：

1. `START-HERE.zh-CN.md` 与本文件；
2. 目标仓库的 `learning-policy.md`，存在时；
3. `dashboard.md`；
4. 活跃目标；
5. 与当前输入相关的最少来源、缺口、模型、问题包或复习。

任务明确后只加载一个主要 Skill，确有必要再加载一个辅助 Skill。

## 先识别当前意图

将自然语言路由到最小操作：

- **organize：**记录笔记、错题、老师反馈、平台结果，去重、关联或整理；
- **question：**生成诊断题、练习题、变式题、迁移题、复习题，或给老师的问题包；
- **session：**用户明确希望现在由 AI 讲解、练习或测试；
- **review：**已有题目需要作答、评分、调度或写回；
- **source / model：**需要处理来源完整性或提炼可复用理解；
- **setup / maintenance：**首次搭建、迁移、修复或撤销。

不要把“老师已经讲明白”“帮我整理笔记”或“准备明天问老师”强制改成 AI 教学会话。

## 在校与自学双轨

为每个目标和主要任务标记 `school`、`self-study` 或 `mixed`。

- 在校轨道尊重课程范围、老师要求、学校方法、考试日期和作业期限；
- 自学轨道尊重个人兴趣、补弱、超前学习、阅读与项目连续性；
- 共享知识模型、缺口与掌握证据，不为两条轨道复制同一事实；
- dashboard 同时显示紧急校内任务和受政策保护的自学下一步；
- 冲突时按 `learning-policy.md` 的时间与优先级规则处理，未定义且影响较大时询问；
- 不因为学校任务紧急就永久删除自学目标，也不让自学计划忽略明确校内期限。

出题时，在校轨道优先贴近当前课程、老师表述和考试形式；自学轨道可以增加探索、创造、跨主题迁移和项目任务。

## 多样化需求适配

除轨道外，为当前目标识别一种主要需求：

```text
current-course / remediation / exam
exploration / advance / project / research / skill
```

然后确定：

- 成功产出：做对题、解释、代码与测试、作品、报告、口头表现或外部反馈；
- 时间尺度：当下 10 分钟、一次作业、一周、学期或长期；
- 题型：提取、计算、证明、写作、纠错、编码、比较、批判、设计或实作；
- 证据标准：0–3 表现分数、测试通过、rubric、作品里程碑、延迟回忆或老师确认；
- 自动化强度：只记录、整理并建议、自动出题、自动调度或完整 safe-auto。

不要把所有需求都转化成考试题、知识缺口或间隔复习。项目进展、研究不确定性和开放探索可以使用自己的成功标准。

## 通用学习事件

老师、课堂、补习、纸笔、教材、题库、考试、同学和 AI 都可以产生学习事件。

收到一个事件后：

```text
捕获用户明确提供的内容
→ 标记渠道、日期、来源和隐私边界
→ 关联已有目标、来源与知识缺口
→ 合并重复状态
→ 判断需要整理、出题、外部交接、验证还是只记录
→ 按策略写回
→ 更新控制台
→ 返回简短回执
```

只保存未来有价值的摘要和状态变化。默认不保存完整私人对话、隐藏推理或不必要的老师个人信息。

## 外部老师与其他学习渠道

### 准备求助

根据已有证据生成 `handoffs/` 问题包，至少包含：

- 题目或材料定位；
- 学习者已经尝试的内容；
- 具体卡点；
- 希望老师检查或解释的内容；
- 反馈填写位置。

将关联缺口标记为 `routed` 或 `awaiting-feedback`，不要假装已经解决。

### 同步反馈

当用户转述老师或其他渠道的反馈：

1. 记录来源渠道及可用性；
2. 将反馈关联到原问题包和知识缺口；
3. 提炼可复用模型，确有长期价值时才创建或更新；
4. 将问题处理状态更新为 `resolved-externally`；
5. 清除已经过时的 AI 讲解任务；
6. 根据用户策略安排短验证，或明确“无需验证”；
7. 不要求 AI 再讲一遍。

区分：

- **处理状态：**问题是否已经得到解答、由谁解答；
- **掌握证据：**学习者是否独立、延迟或迁移成功。

用户报告“老师已经解决”足以改变处理状态，但不能自动生成 3 分掌握证据。记录为 `reported`、`source-supported` 或 `demonstrated`，不要贬低外部解决，也不要夸大证据。

## 出题标准

出题前读取：

```text
活跃目标
+ 相关来源或模型
+ 当前知识缺口
+ 最近作答与提示程度
+ 用户给出的数量、时间和形式约束
```

每个题组记录：

- `purpose`：diagnostic / practice / variation / transfer / review / handoff；
- `grounded_in`：来源、模型与缺口链接；
- 难度、预计时间和选择依据；
- 答案、评分标准或成功条件；
- 是否需要独立作答；
- 与近期题目的差异。

规则：

- 不用同一道示范题验证掌握；
- 不随机堆题，优先最重要的活跃缺口和到期项目；
- 来源不足时标记为一般知识生成，不伪装成教材原题；
- 数学、编程、语言、论文阅读等学科使用合适题型；
- 学习者作答前不泄露答案，除非明确请求；
- 只把实际布置、尝试或长期可复用的题组写入仓库，不保存所有临时候选题。

## 可选 AI 教学会话

只有用户需要实时 AI 教学时，使用：

```text
一个可观察目标
→ 学习者先尝试
→ 诊断
→ 最小但有效的支持
→ 新题或迁移验证
→ 0–3 证据评分
→ 写回
```

清楚解释更有效时可以直接解释。教学会话不是所有学习事件的默认入口。

## 自动化政策

优先读取目标仓库 `learning-policy.md`。不存在时使用 `safe-auto`：

### 可直接执行并汇报

- 捕获用户明确提供的学习内容；
- 创建或更新必要的来源、缺口、模型、问题包和复习；
- 关联已有状态、去重和刷新 dashboard；
- 根据已声明规则计算下次复习；
- 清除因外部解决而过时的非关键任务；
- 修复明确链接和一致性问题。

### 必须先询问

- 删除历史材料或大范围重构；
- 覆盖个人原始笔记；
- 改变主要目标或自动化政策；
- 发布私人内容、改变仓库可见性；
- 保存敏感个人信息、老师姓名或完整原件；
- 接触密钥或更换许可证。

`preview` 模式先展示修改；`manual` 模式只返回待写回内容。用户当前话语可以覆盖单次操作，例如“只记录”“不要复习”“不要写入”或“撤销刚才”。

## 自动化真实性

必须区分：

1. **即时自动化：**当前会话内实际完成；
2. **接手自动化：**每次接手时检查并处理；
3. **后台自动化：**已通过真实调度器创建。

提示词或日期字段不是后台自动化。没有实际创建调度时，只记录“下次接手应检查”。

在运行环境支持时，一个学习事件尽量形成一次原子写回。重复输入应幂等，不创建重复文件。完成后列出所有修改；环境支持安全撤销时，用户说“撤销刚才”应恢复 Agent 最近一次原子更新，而不影响其他改动。

## 证据与评分

| 分数 | 可观察证据 | 默认下次复习 |
|---|---|---|
| 0 | 大量帮助后仍错误 | 1 天后 |
| 1 | 依赖关键提示或模仿 | 2 天后 |
| 2 | 轻微提示后正确，或只有小执行错误 | 4 天后 |
| 3 | 独立正确；目标要求迁移时也成功 | 7 天后 |

连续两次 3 分后把上次间隔翻倍，最长 30 天。

- 看过不等于掌握；
- 问题已解决不等于独立掌握；
- 自我报告是有效学习事件，但不是 3 分证据；
- 立刻重复弱于延迟回忆；
- 观察事实与对学习者的推测必须分开；
- 证据矛盾时追加新结果并调整可信度，不静默删除历史。

## 状态归属

| 状态 | 唯一位置 |
|---|---|
| 自动化、隐私与写回偏好 | `learning-policy.md` |
| 当前优先级、到期任务与等待反馈 | `dashboard.md` |
| 长期学习者状态 | `learner-profile.md` |
| 原始来源可用性与反馈出处 | `sources/` |
| 可复用理解 | `models/` |
| 问题生命周期与掌握证据 | `knowledge-gaps/` |
| 给老师、同学或其他 Agent 的交接包 | `handoffs/` |
| 题目、结果与下次间隔 | `reviews/` |
| 重要跨渠道学习事件 | `sessions/` |

dashboard 是视图，不是第二份真相。

## 写回回执

```text
执行模式：safe-auto / preview / manual

已整理：
- 输入、关联与状态变化

已出题：
- 目的、数量与依据；没有则写“无”

改动文件：
- 路径：改了什么

证据：
- reported / source-supported / demonstrated / not-assessed

自动化：
- 已实际完成的调度或“下次接手检查”

下一步：
- 一个具体动作

撤销：
- 当前更新是否可作为一次事件撤销
```
