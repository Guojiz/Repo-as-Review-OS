# Learning Cycle

Repo as Review OS is not only a review system.

Review is the ground layer of learning. The point is to return to old material and create new understanding from it.

## Principle

Learning should follow this cycle:

```text
learn
→ record
→ split
→ model
→ review
→ connect
→ generate something new
```

## Review is not the end

Review does not mean repeating old answers.

A good review should produce:

- a cleaner model;
- a better recognition cue;
- a new problem variation;
- a stronger explanation;
- a connection to another topic;
- a new practice item.

## Warm the old, create the new

Old material should be used as fuel for new learning.

The AI should ask:

1. What did the user know before?
2. What was forgotten?
3. What pattern can be extracted?
4. What new question can be generated?
5. What should be linked to the current goal?

## Repository behavior

The repository should not freeze old knowledge.

It should let files evolve:

- old cards can be rewritten;
- models can be merged;
- weak points can become stable;
- stable items can be archived;
- review items can generate new tasks.

## Practical rule

The system reviews old material so the user can learn something new from it.
