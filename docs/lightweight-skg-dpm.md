# Legacy Note: Lightweight SKG and DPM

This page is kept only for old links.

Do not use `SKG`, `DPM`, `Static Knowledge Grounding`, or `Dynamic Personal Memory` as GitLearnOS product terminology. Those labels are too close to external tutoring-research language and can make the project look like it is copying or claiming another system's concepts.

GitLearnOS should use plain repository terms instead:

```text
source layer
→ materials, source records, models, evidence, Git history

learner state layer
→ learner-profile.md, knowledge gaps, review results, dashboard, memory notes
```

For the current model, use [Source and Learner State Model](source-and-learner-state.md).

## Rule

External research can be referenced as background, but the product vocabulary should come from the original `zhongkao` learning repository and the GitHub + AI workflow itself.
