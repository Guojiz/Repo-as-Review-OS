# Architecture / 架构

Repo as Review OS has four layers.  
Repo as Review OS 包含四个主要层级。

## 1. Source layer / 来源层

Keeps original materials traceable.  
负责保存并追踪原始材料。

Rule: do not replace a source with a summary.  
规则：不要用摘要替代原始来源。

## 2. Model layer / 模型层

Turns repeated mistakes into reusable patterns.  
把反复出现的错误转化为可复用模型。

Each model should include:  
每个模型应包含：

- recognition cue / 识别信号；
- minimum method / 最小方法；
- common trap / 常见陷阱；
- next action / 下一步动作。

## 3. Handoff layer / 接手层

Explains how future work should continue.  
说明后续编辑者或 AI 工具如何继续工作。

A handoff file should include:  
接手文件应包含：

- what to read first / 先读什么；
- where new material should go / 新材料放哪里；
- what counts as complete / 什么算完成；
- what should stay private / 什么保持私有。

## 4. Deployment layer / 部署层

Turns the general method into a public page or template.  
把通用方法整理成公开页面或模板。
