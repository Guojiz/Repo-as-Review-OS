# 从原系统提炼出的规则

原来的学习仓库说明：Review OS 不能只是模板集合。

它需要真正的运行规则，决定 AI 在每次学习任务前、中、后分别做什么。

## 1. 作答前先检查

在答题、出题或写卡片前，AI 应该先检查：

1. 有没有完整来源？
2. 有没有相似错题、模型或素材卡？
3. 这次产出是否值得写回仓库？

## 2. 状态标签必须诚实

文件不能假装比实际更完整。

可用标签：

- `full-source`
- `draft-full`
- `index`
- `summary-only`
- `not-full-solution`
- `local-only-source`
- `needs-source`
- `todo`

## 3. 先来源，后摘要

如果来源缺失，不要假装已经有完整题卡。

应该先进入待补队列。

## 4. 时间紧时先轻量处理

不是每个项目都要立刻做成完整卡。

冲刺阶段可以先建轻量记录，标明缺什么，之后再回填。

## 5. 明显仓库问题可以直接修

如果 AI 有权限，看到明显问题时可以直接修：

- 断链；
- 过期索引；
- 摘要冒充来源；
- 控制台漏链接；
- 会误导下一个 AI 的旧状态说明。

## 6. 策略变化前要问

这些情况要先问用户：

- 删除重要材料；
- 改变学习策略；
- 发布私人材料；
- 用不符合用户考试或训练规则的方法替代原方法。

## 7. 仓库服务用户

不要为了仓库漂亮而优化仓库。

仓库应该帮助用户学习、记住和行动。

## 实际循环

```text
任务到来
→ 检查目标
→ 检查来源
→ 检查相似模型
→ 作答或生成
→ 有价值就写回
→ 更新索引或控制台
→ 必要时安排复习
→ 汇报改动和缺口
```
