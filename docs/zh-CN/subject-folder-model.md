# 学科文件夹模型

GitLearnOS 使用一个由学习者拥有的仓库，并在其中按学科建立聚焦文件夹。“学科库”指文件夹，不是单独的 GitHub 仓库。

## 标准结构

```text
my-gitlearnos/
├── AGENTS.zh-CN.md
├── learning-policy.md
├── learner-profile.md
├── dashboard.md
├── goals/                       # 可选，只放真正跨学科目标
├── subjects/
│   ├── math/
│   │   ├── goals/
│   │   ├── inbox/
│   │   ├── sources/
│   │   ├── models/
│   │   ├── knowledge-gaps/
│   │   ├── handoffs/
│   │   ├── reviews/
│   │   └── sessions/
│   └── english/
└── archive/
```

这是一套位置规则，不是要求一次创建全部目录。Git 不保存空文件夹；Agent 只建立当前活跃学科和当前事件需要的文件。

## 根目录与学科归属

| 根目录 | 学科文件夹 |
|---|---|
| 自动化与隐私政策 | 学科目标与约束 |
| 稳定学习偏好 | 学科来源与外部反馈 |
| 跨学科优先级视图 | 模型与知识缺口 |
| 真正跨学科的目标 | 题目、结果、问题包与活动 |

根目录 dashboard 链接学科状态，不把它复制成第二份真相。

## 自动路由

按以下顺序确定学科：

1. 用户明确说明；
2. 已有关联目标或路径；
3. 材料中的清晰学科线索；
4. 只有歧义会改变写入位置时，问一个简短问题。

自然语言纠正覆盖最近一次路由。一个事件确实跨学科时，选择一个主要归属，并链接次要学科，不复制整条记录。

## 命名

优先使用稳定的小写名称，如 `math`、`english`、`physics`、`coding`。已有清晰用户习惯时复用；大范围改名前先询问。

## 迁移

对于已有单科学习仓库：

1. 识别当前活跃学科；
2. 新状态立即写入 `subjects/<subject>/`；
3. 只有能安全保护路径和链接时才逐步移动旧文件；
4. 大范围移动前询问；
5. 新旧位置暂时并存时留下简短迁移说明。

目标是减少上下文和维护成本，不是追求完美目录树。
