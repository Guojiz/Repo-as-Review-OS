# 运行环境自适应

GitLearnOS 保存可移植政策与状态，当前主 AI 提供执行能力。

## 检测能力，不只看品牌

```text
能读模板？
能读写目标仓？
能访问授权来源？
能加载文件型 Skills 或仓库规则？
能原子写入或访问版本历史？
有原生记忆？
有真实调度器？
哪些步骤确实必须由用户完成？
```

安全时直接测试。产品名称不能证明权限。

## 工具型 Work 路径

在 ChatGPT Work 等已连接工作区中：

```text
验证目标与政策
→ 检查当前状态
→ 分类学习事件
→ 整理或生成针对性问题
→ 直接写入安全变化
→ 验证并返回回执
```

不要让用户粘贴仓库、重建文件或完成当前环境已经能够完成的操作。

## 仓库 Agent 路径

Codex、Claude Code、Cursor等应：

- 遵守仓库指令；
- 检查已有改动；
- 保护无关工作；
- 使用同样的整理、出题、自动化和证据规则；
- 保护私人原件；
- 汇报验证与改动文件。

使用一个主 Agent。搜索、代码执行和可视化只是能力，不是独立学习状态拥有者。

## 只读路径

没有写权限时：

- 处理当前输入或生成题目；
- 只读取真正可用的来源；
- 返回精确待写回区块；
- 明确仓库没有改变；
- 将稳定偏好与快速变化状态分开。

## 三层自动化

### 即时

当前交互内真实完成整理、出题与写回。

### 接手

每次 Agent 恢复时检查：

- 未处理 inbox；
- 到期题目；
- 等待外部反馈；
- 已解决问题的过时任务；
- 当前唯一下一步。

### 后台

只有 Runtime 暴露真实调度器时才创建。

```text
文件里的日期或说明 ≠ 已调度自动化
真实调度任务并验证状态 = 已调度自动化
```

基础套件保证即时与接手自动化，后台执行是可选适配。

## 政策与安全

读取 `learning-policy.md`。`safe-auto` 下普通、低风险、可逆状态变化无需重复确认。删除、大范围覆盖、政策变化、公开状态、私人发布、敏感身份、密钥和许可证变化前询问。

重复输入应幂等。尽量一次事件一次原子更新，并汇报能否安全撤销。

## 规则

适配执行方式，不降低产品保证。所有 Runtime 都应保持跨渠道来源、针对性出题、证据诚实、获授权时自动写回和用户掌控。
