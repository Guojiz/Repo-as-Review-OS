# 动态规则

Repo as Review OS 不应该把所有规则都当成死规则。

学习系统会随着用户目标、时间压力、材料完整度和当前阶段变化。

## 核心想法

原则稳定，策略可调。

稳定原则：

- 来源可追踪；
- 不编造缺失材料；
- 文件要服务目标；
- 改动要汇报；
- 本地材料要保护；
- GitHub 是操作层。

可调整策略：

- 卡片写多详细；
- 现在建完整卡，还是先进入待补队列；
- 优先速度，还是优先完整度；
- 归档、合并还是重写；
- 多久复习一次；
- 先运行哪个自动化。

## 阶段感知

AI 应该识别当前阶段。

### 冲刺阶段

时间紧时，优先有用输出，不追求完美结构。

可以先建轻量卡，标明缺什么，之后再回填。

### 建设阶段

时间充足时，把轻量记录扩展成完整题卡、模型、控制台和复习循环。

### 维护阶段

系统稳定后，重点整理、安排复习、发现过期内容。

## 材料完整度

来源完整，就建完整卡。

来源不完整，就进入待补队列，并标明缺什么。

用户只记得线索时，可以保存线索，但不能当完整来源。

## 实用规则

不要为了仓库漂亮牺牲学习效率。

仓库服务目标，不服务自己。
