# Skill、记忆与状态

GitLearnOS 将行为、长期状态和临时上下文分开。

```text
Skill 或仓库规则
→ Agent 应该怎样行动

目标仓库
→ 当前可检查学习状态

AI 原生记忆
→ 稳定偏好与长期模式的可选缓存

当前对话或工作区
→ 临时执行空间
```

## 具备工具能力的 Work 路径

当工作区能读取来源并写入目标仓库时：

1. 加载 Router 或统一 Agent 规则；
2. 只检查活跃目标状态；
3. 直接完成学习任务；
4. 将有证据支撑的变化写入唯一状态文件；
5. 只有稳定偏好或长期模式改变时，才建议更新原生记忆。

不能让原生记忆成为学习状态的唯一副本。当前获授权 Runtime 能修改文件时，也不要让用户手工复制。

## Skill 路径

从这里开始：

```text
skills/repo-as-review-os/SKILL.md
```

然后只加载一个具体子 Skill。Skill 保存行为，不保存持续变化的个人状态。

## 仓库指令路径

如果不支持自定义 Skills，使用 `START-HERE.zh-CN.md` 与 `AGENTS.zh-CN.md` 作为可执行约定。不要用“读取全部文档”来补偿。

## 原生记忆

适合保存：

- 稳定语言与讲解偏好；
- 长期目标；
- 长期隐私限制；
- 多次证据支持的重复模式；
- 选择哪个目标仓库作为唯一状态的规则。

默认不保存：

- 一次性错误；
- 到期日期与临时任务；
- 原始私人笔记；
- 不完整来源；
- 未验证假设；
- 完整复习历史。

## 冲突规则

原生记忆与仓库状态不一致时：

1. 检查时间与证据；
2. 确认哪一层过时；
3. 明确更新；
4. 无法判断时记录不确定性，不能静默选择方便的版本。

## 回退

单上下文或只读环境只提供一次任务需要的文件，并输出待写回区块。写入目标仓库后再恢复长期连续性。
