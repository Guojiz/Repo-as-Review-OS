# 自动化模型

Repo as Review OS 的自动化应该把仓库当作状态来源。

AI 不应该只凭记忆出题或整理，而应该先读取仓库文件、更新时间、控制台和上一次输出。

## 自动化 1：仓库整理器

目的：让仓库保持可用。

定时执行：

1. 读取控制台和索引；
2. 找出不完整文件；
3. 找出过期 TODO；
4. 更新控制台；
5. 汇报改动。

输出可以写入：

```text
automations/repository-organizer/YYYY-MM-DD.md
```

## 自动化 2：题单生成器

目的：根据仓库历史生成复习任务。

定时执行：

1. 读取最近题目卡；
2. 读取旧薄弱点；
3. 读取活跃模型；
4. 检查哪些内容最近没有出现；
5. 生成小型复习题单；
6. 写回仓库。

输出可以写入：

```text
automations/practice-generator/YYYY-MM-DD.md
```

## Git 历史的作用

Git 历史帮助 AI 判断新旧。

AI 可以根据文件更新、提交历史和控制台记录判断什么是新的、什么很久没复习、什么应该回炉。

## 边界

定时任务应该整理仓库、生成题单、汇报改动。

删除文件、公开私人记录、上传本地材料之前必须先确认。
