# 为什么使用 GitHub

Repo as Review OS 使用 GitHub，不是因为 GitHub 只是一个网盘，而是因为它是一条可以写入的时间线。

## 1. 每次变化都有时间和历史

普通文件夹只能保存文件。GitHub 会记录文件什么时候变了、变了什么、为什么变。

这对学习很重要。每一道题、每一次订正、每一个摘要、每一次模板更新，都会变成可追踪的学习轨迹。

## 2. pull 和 push 形成学习节奏

仓库可以被读取、更新、再写回。

对 AI 来说，流程是：

1. 读取仓库最新状态；
2. 解题或分析任务；
3. 把结果写回；
4. 留下改动记录；
5. 下次继续利用这些记录。

## 3. 题目可以变成模型

用户每做一道数学题，AI 不只是写答案。

它可以保存原题、解法路径、错误原因、识别信号、共同模型和未来复习项。

时间久了，单个题目就会变成可复用模型。

## 4. 可以按时间复习

因为文件和提交有历史，AI 可以判断：

- 最近新增了什么？
- 哪些内容很久没复习？
- 哪些错误反复出现？
- 下一份题单应该出什么？

## 5. 为什么不只用 AI 项目

AI 项目可以保存指令和上下文，但它本身不够。

GitHub 给 AI 一个开放文件树、结构化写回、版本历史和持久工作区。

## 6. 为什么不使用很重的本地方案

Obsidian、本地文件夹、命令行 Agent 和桌面编辑器都可以很强，但它们会增加重量。

对很多人来说，最方便的形态是：

```text
一个 AI App
+ 一个 GitHub 仓库
+ 一个连接器
+ 一层记忆或项目规则
```

它可以在手机端运行，不需要用户维护完整本地工具链。
