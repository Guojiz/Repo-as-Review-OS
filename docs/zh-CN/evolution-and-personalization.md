# 演化与个人化

Repo as Review OS 不是把用户所有资料一股脑丢进仓库。

它是一个会变化的个人系统。AI 应该拆解材料、提取规律、整理结构、重写内容，并在确认后移除不再有用的部分。

## 不是资料垃圾桶

不要把所有东西直接复制进仓库。

AI 应该按这个流程处理材料：

```text
原始材料
→ 来源记录
→ 提取题目或想法
→ 可复用模型
→ 复习项目
→ 更新控制台
```

## 核心动作

AI 应该能够：

- 把大材料拆成小单元；
- 在保留来源状态后再摘要；
- 提取共同规律；
- 合并重复笔记；
- 标记薄弱或不完整内容；
- 重写混乱文件；
- 确认后删除或归档无用结构；
- 根据用户习惯调整系统。

## 个人化

没有一种固定目录适合所有用户。

Review OS 应该适配：

- 用户学科；
- 考试或训练目标；
- 偏好语言；
- 可用 AI 工具；
- 权限等级；
- 本地材料情况；
- 日常复习节奏。

## 没有什么是神圣不可改的

模板、目录、控制台和规则都可以改。

真正稳定的是这些原则：

1. 来源可追踪；
2. 区分来源、摘要、模型、索引和待补；
3. 本地材料受保护；
4. GitHub 是操作层；
5. AI 汇报改动。

## AI 行为

AI 不应该要求用户一开始就设计完整系统。

它应该先搭一个小而能用的结构，再随着真实材料进入不断调整。

## 实用规则

不确定时，不要先增加更多内容。先拆解、合并、标记或简化。
