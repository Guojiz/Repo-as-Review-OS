# 学习循环

Repo as Review OS 不应该只是复习系统。

复习是学习的底层动作。真正的目的，是回到旧材料，并从旧材料里长出新的理解。

## 原则

学习应该形成这个循环：

```text
学习
→ 记录
→ 拆解
→ 建模
→ 温习
→ 连接
→ 生成新东西
```

## 复习不是终点

复习不是重复旧答案。

好的复习应该产生：

- 更清楚的模型；
- 更准确的识别信号；
- 新的变式题；
- 更强的解释；
- 和其他主题的连接；
- 新的练习任务。

## 温故而知新

旧材料应该成为新学习的燃料。

AI 应该问：

1. 用户以前懂了什么？
2. 什么又忘了？
3. 可以提取出什么规律？
4. 可以生成什么新问题？
5. 应该和当前目标建立什么连接？

## 仓库行为

仓库不应该把旧知识冻住。

它应该允许文件不断演化：

- 旧卡片可以重写；
- 模型可以合并；
- 薄弱点可以变稳定；
- 稳定项目可以归档；
- 复习项目可以生成新任务。

## 实用规则

系统温习旧材料，是为了让用户从中学到新东西。
