# 间隔复习模型

Repo as Review OS 应该包含间隔复习机制。

目标很简单：重要知识应该在遗忘之前被重新带回来。

## 为什么重要

学习系统不应该只是保存用户曾经学过什么，还应该在合适的时间把旧内容重新推回来。

这样可以避免一种常见失败：用户曾经听懂了一道题，但几个月后又忘了。

## 复习周期

一个简单默认周期：

```text
第一次学习
→ 当天
→ 第二天
→ 3 天后
→ 7 天后
→ 14 天后
→ 30 天后
→ 目标截止前
```

这个周期可以按用户情况调整。

## 哪些内容要进入周期

AI 应该给这些内容安排复习：

- 错题模型；
- 重要题目卡；
- 薄弱主题；
- 词汇；
- 写作素材；
- 公式；
- 反复出现的陷阱；
- 与目标强相关的知识。

## 仓库字段

每个需要复习的项目可以包含：

```text
First learned:
Last reviewed:
Next review:
Review count:
Memory status:
Goal link:
```

## 记忆状态

推荐状态：

- `new`
- `learning`
- `unstable`
- `stable`
- `forgotten`
- `goal-critical`

## AI 行为

AI 看到一个可复习项目时，应该判断：

1. 它是否绑定目标？
2. 上次复习是什么时候？
3. 它稳定了，还是已经遗忘？
4. 它是否应该进入下一份题单？
5. 是否需要更新下一次复习日期？

## 自动化

题单生成器应该使用间隔复习。

它不应该只挑最近内容，也应该把到期的旧内容重新捞出来。

## 实用规则

一道题不是做对一次就结束。

只有隔了一段时间之后，用户仍然能识别并解决它的模型，它才算真正稳定。
