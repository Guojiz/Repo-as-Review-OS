# 基于证据的间隔复习

GitLearnOS 根据实际表现安排复习，而不是根据文件创建时间，或学习者是否看过一次讲解。

本协议适用于需要经过时间后再次提取或执行的知识与技能。不要把开放研究、探索阅读、创作或项目里程碑强制塞进 0–3 间隔复习；它们应使用目标声明的 rubric、测试、作品或 milestone 证据。

## 证据评分

在一道新检验后给出一个分数：

| 分数 | 可观察结果 | 默认间隔 |
|---|---|---|
| 0 | 大量帮助后仍不能正确回忆 | 1 天 |
| 1 | 依靠关键提示或模仿才能正确 | 2 天 |
| 2 | 只需轻微提示，或仅有小执行错误 | 4 天 |
| 3 | 独立正确；要求迁移时也成功 | 7 天 |

连续两次得到 3 分后，把上次间隔翻倍，最长 30 天。之后如果得到更低分，根据新分数重新开始计算。

## 什么算证据

较强证据：

- 延迟后的独立回忆；
- 用自己的语言正确解释；
- 完成变式；
- 成功迁移到新情境；
- 自己发现并纠正错误。

较弱或不能证明掌握的证据：

- 阅读答案；
- 照抄示例；
- 立刻重复；
- 做完一个文件或题单；
- 没有检验就说“我懂了”。

弱证据也应诚实记录，它可以指导后续支持，但不能支持 3 分或 `stable`。

## 复习记录字段

每次已评分复习应包含：

```text
复习项目：
日期：
题目或任务：
学习者回答摘要：
提示程度：无 / 轻微 / 关键 / 完整讲解
正确性：
是否要求迁移：是 / 否
迁移结果：
分数：0 / 1 / 2 / 3
上次间隔：
下次复习：
证据链接：
```

## 项目状态

保持简单：

- `new`：尚未评估；
- `learning`：证据为 0–2 分，或只有即时证据；
- `stable`：至少两次成功复习，并包含延迟独立证据；
- `reopened`：曾经稳定，但后来再次失败；
- `blocked`：缺少来源或前置知识，无法有效检验。

`goal-critical` 是优先级，不是记忆状态。

## 复习选择顺序

1. 已逾期且目标关键的项目；
2. 最近得到 0 或 1 分的项目；
3. 到期的活跃知识缺口；
4. 需要迁移检验的 2 分项目；
5. 间隔已到的稳定项目。

一次会话应小到能够完成并评分。只生成、未作答的复习保持 `planned`。

## 日期规则

使用明确的 ISO 日期（`YYYY-MM-DD`），并记录是哪一个分数产生了该日期。如果当前环境不能计算或写入，明确标记为待处理，不能假装后台自动化会完成。

## 原则

对于需要保持的知识和可重复技能，做对一次不等于稳定掌握；需要之后再次提取或表现。其他学习需求使用各自声明的证据标准。
