# 与原始学习仓库的对齐

原始 `zhongkao` 仓库是 GitLearnOS 的实践落实基线，用于检验轻量文件工作流在真实学习中是否仍然有用。

## 必须保留的有效行为

1. 一个由学习者拥有、按学科分文件夹的长期仓库；
2. 来源、摘要与缺失材料诚实分开；
3. 将错题转化成可复用识别模型；
4. 旧内容经过时间后重新出现；
5. 控制台给出一个具体下一步；
6. 另一个有能力的 Agent 能从可检查状态继续；
7. 默认保护本地与私人材料；
8. 学习者自己能看懂的小型文件。

## GitLearnOS 加强的行为

- 老师、课堂、纸笔、平台与 AI 的跨渠道捕获；
- 自动整理、关联与去重；
- 个性化学习题目与外部答疑问题包；
- 用户拥有的自动化政策与透明写回回执；
- 问题处理状态与掌握状态分离；
- 学习画像证据链接；
- 知识缺口类型与可信度；
- 作出掌握判断前先让学习者尝试；
- 提示程度记录；
- 0–3 可观察证据评分；
- 迁移与延迟回忆检验；
- 可复现的下次复习日期；
- 选择性学习事件写回。

## GitHub 本身不能提供什么

GitHub 不会自己整理、出题、检查不可访问的原件、运行 AI 模型或保证后台执行。一个当前主 AI Runtime 在获授权时提供这些能力。

## 对齐检验

无需另做完整平台，学习者就能完成以下闭环时，才算通过实践对齐：

```text
材料、错误或外部反馈
→ 自动整理与关联
→ 针对性学习题或老师问题包
→ 反馈或学习者结果
→ 可复用状态
→ 自动写回与下一步
```

只创建文件夹不算对齐。
