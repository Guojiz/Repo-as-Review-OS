# 部署指南

GitLearnOS 的“部署”是把套件挂载到一个由学习者拥有的目标仓库，不是安装另一套完整教学平台。

## 快速路径：具备工具能力的工作区

当当前 AI 能读取模板并写入目标仓库时使用；已获授权的 ChatGPT Work 会话就是一种例子。

```text
1. 确认目标仓库与活跃目标
2. 通过现有工具验证读写能力
3. 检查目标仓库已有文件
4. 阅读 START-HERE.zh-CN.md 与 AGENTS.zh-CN.md
5. 只创建缺少的最小状态
6. 进行第一次学习会话
7. 验证并汇报写回
```

首次最小状态：

```text
AGENTS.zh-CN.md
dashboard.md
learner-profile.md
goals/main-goal.md
```

真实学习产生第一份来源、模型、缺口、复习或会话状态时再创建对应文件。默认不要把所有模板文档、Demo、平台指南和 Skills 全部复制进目标仓库。

## 已有目标仓库

- 修改前先检查；
- 保留无关文件；
- 先映射一条活跃学习路径；
- 大范围移动、覆盖或删除前询问；
- 现有文件已经承担某项状态时，不要再创建平行状态系统。

## 只读回退

输出精确 Markdown 写回区块，并明确标为待处理。学习者可以手工应用，或之后在具备工具能力的会话中写入。

## 选择 Runtime

按能力选择：

| 能力 | 路径 |
|---|---|
| 已连接目标仓读写与来源工具 | Work 直接执行路径 |
| 本地或云端仓库 checkout | 仓库 Agent 路径 |
| 能教学或读上下文，但不能写入 | 只读接手路径 |
| 只有粘贴摘录 | 单任务回退 |

只要具备所需能力且用户选择，ChatGPT、Codex、Claude、Cursor 或其他 Agent 都可以成为主 Runtime。OpenHanako 是可选桌面适配器，不是默认依赖。

## 成功检验

只有 Agent 能识别活跃目标、完成一次先尝试后讲解的学习会话、记录可观察证据、更新正确状态文件并汇报一个下一步时，才算部署完成。

只创建文件夹不算部署成功。
