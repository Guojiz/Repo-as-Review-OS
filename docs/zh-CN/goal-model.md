# 目标模型

Repo as Review OS 必须是目标驱动的。

AI 不应该一开始就把旧材料全部倒进仓库。它应该先理解用户的目标，再决定什么要保留、拆解、重写、生成、复习或归档。

## 为什么目标重要

没有目标的仓库会变成资料堆。

有目标的仓库才会变成操作系统。

目标可以是：

- 准备一次考试；
- 提高数学解题能力；
- 建立写作素材系统；
- 训练词汇；
- 准备研究项目；
- 维护长期学习记录。

## 目标层级

目标应该分层保存：

```text
主目标
→ 子目标
→ 薄弱点
→ 来源材料
→ 模型
→ 复习任务
→ 自动化输出
```

## 示例

如果主目标是考试准备，AI 可以创建：

- 学科目标；
- 薄弱主题清单；
- 题目模型；
- 每日复习计划；
- 生成题单；
- 进度记录。

AI 不应该只是把旧材料还给用户，而应该在旧材料基础上创建新结构。

## 目标文件

每个 Review OS 可以包含：

```text
goals/main-goal.md
goals/subgoals.md
goals/progress.md
```

## AI 行为

整理材料前，AI 应该先判断：

1. 用户当前主目标是什么？
2. 时间范围是什么？
3. 哪些学科或能力最重要？
4. 已经暴露了哪些薄弱点？
5. 下一步应该生成什么？
6. 什么应该归档或降低优先级？

## 在旧材料基础上创新

旧材料是原矿，不是成品。

AI 应该把它转化为：

- 模型；
- 清单；
- 题单；
- 控制台；
- 复习循环；
- 可复用模板。

## 实用规则

每个文件都应该回答一个问题：

**它服务于哪个目标？**

如果它不服务当前或未来目标，就归档，或标记为低优先级。
