# 什么内容放进 GitHub

Repo as Review OS 把 GitHub 当作结构化学习记录的位置。

这不等于每一个原始文件都必须复制进仓库。

## 适合放进仓库的内容

- 学习目标
- 来源记录
- 摘要
- 可复用模型
- 复习题单
- 控制台
- 下次复习日期
- AI 接手记录

这些文件描述学习过程，方便 AI 下次继续接手。

## 可以留在外面的内容

- 很大的文件
- 原始截图
- 教材页面
- 本地工作文件
- 其他软件导出的原始文件

如果文件留在 GitHub 外面，就在仓库里写一条来源记录，说明它在哪里，以及 AI 已经看过哪一部分。

## 简单例子

```text
把一道数学题发给 AI。
AI 写入来源记录。
AI 提炼可复用方法。
AI 加入复习项。
AI 更新控制台。
```

## 基础使用方式

基础版可以从这里开始：

```text
ChatGPT Project
+ GitHub 仓库
+ 本地科目文件夹
```

或者：

```text
Codex 或其他 coding agent
+ 本地工作区
+ 目标学习仓库
```

GitHub Actions 和 API key 是可选高级层，不是首次部署必需项。

## 一句话回答

可以。Repo as Review OS 可以把学习数据整理进 GitHub，但 GitHub 主要保存结构化学习记录，不一定保存每个原始文件。
