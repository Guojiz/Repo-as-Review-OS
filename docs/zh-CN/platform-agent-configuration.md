# Agent Runtime 配置

根据能力配置 GitLearnOS，不按产品品牌划分高低。

## 必须遵守的约定

唯一活跃主 Agent 应遵循：

```text
一个目标仓库与 learning-policy
→ 整理任意学习渠道
→ 生成针对性问题
→ 自动执行安全写回
→ 分离问题处理与掌握证据
→ 一个明确下一步与回执
```

## 能力矩阵

| Runtime 能力 | 配置 |
|---|---|
| 仓库与连接来源工具 | 直接执行；先验证再写入 |
| 仓库 checkout 与文件工具 | 遵守本地指令并保护工作区 |
| 有原生记忆但不能写入 | 整理或教学，返回待写回区块 |
| 没有稳定记忆或文件 | 使用粘贴状态完成一个聚焦任务 |

## ChatGPT Work 示例

```text
状态：私有 GitHub 目标仓库
行为：GitLearnOS Router 或 AGENTS.zh-CN.md
来源：只有获授权时使用连接文件或应用
记忆：只缓存稳定偏好
写回：目标仓库唯一状态文件
```

Agent 应直接使用现有工具，自己能安全完成的操作不能再要求学习者重复。

## 仓库 Agent 示例

如果 Codex、Claude Code、Cursor 或类似 Agent 是用户选择的主 Runtime，它可以直接运行 GitLearnOS。它应读取仓库指令、检查已有改动，并使用同样的学习证据标准。

## 单一主 Agent 标准

ChatGPT Work 等工具型 Runtime 能在一个工作区中整理、出题、需要时教学、写入并验证。不要把这些职责拆成多个 Agent。真实调度器可以扩展后台执行，但唯一状态仍在目标仓库。

## 记忆指令

```text
将目标仓库作为 GitLearnOS 唯一状态。原生记忆只保存稳定偏好与长期模式。原生记忆与仓库证据冲突时，检查时间并明确更新过时的一层。
```

## 诚实汇报

必须汇报真实读写与来源权限。写出的提示词不是自动化，提出的补丁不是仓库更新，生成的题组在学习者作答前也不是学习证据。
