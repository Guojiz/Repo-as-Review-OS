# 本地与云端分工

Repo as Review OS 应该区分：什么放在用户本地，什么放进 GitHub。

## 核心原则

GitHub 是共享操作层，本地存储是受保护的来源层。

## 放在 GitHub 的内容

GitHub 适合保存：

- README；
- AGENTS 接手入口；
- 接手规则；
- 控制台；
- 索引；
- 模板；
- 清理后的示例；
- 来源状态说明；
- 本地材料的引用；
- 生成的复习计划；
- 公开网站文案。

GitHub 负责结构、状态和可复用方法。

## 留在本地的内容

本地适合保存：

- 原教材；
- 有版权的 PDF；
- 原始扫描件；
- 私人截图；
- 老师文件；
- 私人笔记；
- 大型媒体文件；
- AI 无法安全访问的文件。

本地负责受保护的原始来源。

## 如何引用本地材料

不要默认上传私人或有版权的来源。

可以在 GitHub 里创建来源记录：

```text
Status: local-only-source
Local reference: 教材章节、页码或文件名
Public summary: 简短安全说明
Needed action: 如果 AI 需要，用户再上传必要摘录或截图
```

## AI 行为规则

AI 应该：

1. 把 GitHub 当作主地图；
2. 检查所需来源是在 GitHub 上，还是只在本地；
3. 只在需要时请求用户上传局部摘录；
4. 不能假装读过自己无法访问的本地文件；
5. 把清理后的笔记、模型和索引写回 GitHub。

## 单一 AI 流程

```text
用户 → ChatGPT → GitHub 仓库
             ↘ 需要时才请求用户上传本地摘录
```

不需要额外本地 Agent。

## 与旧系统的区别

旧系统曾经依赖本地 Agent 巡检文件、拉取任务、推送结果。这个模板默认主 AI 是 ChatGPT 或类似单一 AI。因此，本地材料需要用户在必要时主动提供。
