# What Goes Into GitHub

Repo as Review OS uses GitHub as the place for organized learning records.

It is not a rule that every original file must be copied into the repository.

## Put these in the repository

- goals
- source records
- summaries
- reusable models
- review sets
- dashboards
- next review dates
- agent handoff notes

These files describe the learning process and make it easy for an AI tool to continue later.

## Keep these outside when needed

- very large files
- original screenshots
- textbook pages
- local working files
- raw exports from other apps

When a file stays outside GitHub, add a source record that says where it is and what part the AI has already seen.

## Simple example

```text
A math problem is shown to the AI.
The AI writes a source record.
The AI extracts the reusable method.
The AI adds a review item.
The dashboard is updated.
```

## Basic setup

Basic use can start with:

```text
ChatGPT Project
+ GitHub repository
+ local subject folders
```

or:

```text
Codex or another coding agent
+ local workspace
+ target learning repository
```

GitHub Actions and API keys are optional advanced layers, not required for the first setup.

## One-sentence answer

Yes, Repo as Review OS can organize learning data into GitHub, but GitHub mainly stores the structured learning records rather than every raw original file.
