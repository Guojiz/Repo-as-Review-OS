# 公开前检查清单

这份清单用于把 Repo as Review OS 发到朋友圈、学校群聊、研究群或开源社区前快速检查。

## 1. 定位是否清楚

- [ ] 是否说明它是“基于 GitHub 的目标驱动型 AI 学习操作系统”？
- [ ] 是否避免把它说成普通资料库？
- [ ] 是否说明它不是 ChatGPT 专属，也不是 Obsidian 专属？
- [ ] 是否说明当前是 public alpha / 早期公开版本？

## 2. 入口是否清楚

- [ ] README 是否能在第一屏讲清楚？
- [ ] 是否有“复制给 AI 的提示词”？
- [ ] 是否链接了 `OPERATING-MODEL`？
- [ ] 是否链接了 demo？
- [ ] 是否链接了贡献指南？

## 3. 安全是否清楚

- [ ] 是否提醒真实学习资料用私有仓库？
- [ ] 是否提醒不要公开老师文件、教材、私人截图、真实考试记录？
- [ ] 是否说明本地材料只在需要时提供最小摘录？
- [ ] 是否说明 AI 不能假装读过无法访问的本地文件？

## 4. 贡献路径是否清楚

- [ ] 是否有 `CONTRIBUTING.md`？
- [ ] 是否有 `CONTRIBUTING.zh-CN.md`？
- [ ] 是否有 issue 模板？
- [ ] 是否说明可以贡献 demo、模板、翻译、自动化提示词和反馈？

## 5. demo 是否清楚

- [ ] 是否有一个最小可读 demo？
- [ ] demo 是否不含真实私人资料？
- [ ] demo 是否展示目标、模型、题单、控制台和复习安排？

## 6. 发出去时怎么说

推荐文案：

```text
我做了一个 public alpha 项目：Repo as Review OS。

它是一个基于 GitHub 的目标驱动型 AI 学习操作系统，不是普通资料库。

核心想法是：让 AI 围绕目标，把学习材料拆成来源、模型、题单、间隔复习和自动化学习循环。GitHub 提供文件系统、写回、版本历史和学习时间线。

目前有一个中文轻量 demo，可以直接把仓库丢给 AI，让它解释并尝试帮你搭建自己的学习 OS。

欢迎提 issue、贡献 demo、模板、自动化提示词或使用反馈。
```

## 7. 当前建议

可以公开，但请标明：

```text
public alpha / 早期公开版本
```

不要包装成最终成品。
