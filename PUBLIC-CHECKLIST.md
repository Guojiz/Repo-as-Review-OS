# Public Release Checklist

Use this checklist before sharing Repo as Review OS in social feeds, school groups, research groups, or open-source communities.

## 1. Positioning

- [ ] Does the project explain that it is a goal-driven AI learning operating system built on GitHub?
- [ ] Does it avoid sounding like a normal file archive?
- [ ] Does it state that it is not tied to one AI product or one note-taking app?
- [ ] Does it clearly say public alpha?

## 2. Entry points

- [ ] Can the README explain the project in the first screen?
- [ ] Is there a prompt users can copy into an AI tool?
- [ ] Is `OPERATING-MODEL.md` linked?
- [ ] Is the demo linked?
- [ ] Is the contribution guide linked?

## 3. Safety

- [ ] Does it remind users to use private repositories for real study records?
- [ ] Does it warn against publishing teacher files, textbooks, screenshots, private notes, or real exam records?
- [ ] Does it explain that local material should only be shared as small excerpts when needed?
- [ ] Does it state that AI should not claim to have read local files it cannot access?

## 4. Contribution path

- [ ] Is there a `CONTRIBUTING.md`?
- [ ] Are there issue templates?
- [ ] Does it say people can contribute demos, templates, translations, automation prompts, and feedback?
- [ ] Does it explain what must not be contributed?

## 5. Demo

- [ ] Is there a minimal readable demo?
- [ ] Does the demo avoid private material?
- [ ] Does the demo show goal, model, practice set, dashboard, and review schedule?

## 6. Suggested announcement

```text
I am releasing Repo as Review OS as a public alpha.

It is a goal-driven AI learning operating system built on GitHub, not a normal study archive.

The core idea is to let AI organize learning around goals, sources, models, practice sets, spaced repetition, and automation. GitHub provides the file system, writeback, version history, and learning timeline.

There is a small demo in the repository. You can give the repo to an AI tool and ask it to explain or help build your own learning OS.

Feedback, demos, templates, automation prompts, and issue reports are welcome.
```

## 7. Current recommendation

It is ready to share as:

```text
public alpha
```

Do not present it as a finished product.
