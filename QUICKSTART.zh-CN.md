# 快速开始

当当前 AI 工作区能读取模板仓库，并能写入一个私有目标仓库时，GitLearnOS 的使用最简单。

## 快速路径：具备工具能力的工作区

适用于 ChatGPT Work、能操作仓库的编程 Agent，或其他已连接 GitHub 写入权限的环境。

1. 创建或选择一个私有目标仓库。
2. 可以时只连接这一个仓库。
3. 告诉 Agent 目标仓库链接与一个学习目标。
4. 让 Agent 自己检查、搭建并完成第一次学习会话。

提示词：

```text
把 https://github.com/Guojiz/Repo-as-Review-OS 作为 GitLearnOS 模板仓库。
目标仓库：<链接>
学习目标：<目标>

先读 START-HERE.zh-CN.md 与 AGENTS.zh-CN.md。验证真实权限，检查并保护目标仓库已有文件，只创建缺少的最小学习状态。如果支持 Skills，使用 skills/repo-as-review-os/SKILL.md。然后进行一次短学习会话，只写回有证据支撑的变化，并逐项汇报改动文件。
```

Agent 自己能安全创建的文件，不应再要求你手工重建。

## 回退路径：聊天或只读环境

如果 AI 无法写入 GitHub：

1. 粘贴活跃目标、控制台、学习画像，以及本次任务相关的来源、模型或缺口摘录；
2. 只完成一个明确任务；
3. 让 AI 输出可写回区块；
4. 手工写入目标仓库，或在下一次具备工具能力的会话中写入。

AI 必须说明“尚未写回”，不能声称仓库已经更新。

## 最小首次搭建

不要把整个公开模板复制进个人仓库。先建立：

```text
AGENTS.zh-CN.md
dashboard.md
learner-profile.md
goals/main-goal.md
```

然后创建第一份真正相关的来源、知识缺口、模型、复习或会话文件。只有存在真实状态时才创建目录；Git 不保存空目录。

## 第一次可以这样用

```text
检查我的控制台，直接进行今天最重要的 20 分钟学习会话。
```

```text
诚实记录这份材料，先测试我能回忆多少，再开始讲解。
```

```text
把这次错误记录成知识缺口和可复用模型，再安排一道迁移检验。
```

```text
根据实际证据为这次复习评分，并更新下次复习日期。
```

## 搭建成功的检验

Agent 应能带着文件链接回答：

- 当前目标是什么？
- 现在什么内容到期？
- 哪些知识缺口有实际证据？
- 学习者刚刚真正证明了什么？
- 下一步是什么？
- 修改了哪些文件？

如果它只能列出文件夹，却不能进行并记录一次学习会话，搭建仍未完成。

## 隐私

- 真实学习记录使用私有目标仓库。
- 不要把完整版权教材、老师文件、原始私人截图、凭据或敏感个人材料放进公开 GitHub。
- 只有获授权时才使用连接或本地原件，并且只写回最小、长期有价值的学习状态。
