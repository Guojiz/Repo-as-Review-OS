# 从这里开始：Agent 接手入口

GitLearnOS 是用户拥有的学习控制层。一个主 Agent 负责自动整理、针对性出题和安全写回；AI 实时教学只是可选路径。不要预加载整个模板仓库。

## 1. 识别两个仓库

```text
模板仓库：Guojiz/Repo-as-Review-OS
目标学习仓库：<用户仓库或未知>
```

绝不能把个人学习数据写进模板仓库。如果任务需要写回但目标仓库未知，只询问这一项。

## 2. 检测真实能力

在内部确认：

```text
模板读取：
目标仓读取：
目标仓写入：
连接来源：
Skill 支持：
真实调度支持：
必须由用户完成的步骤：
```

当前环境能安全完成的事情，不要让学习者手工重做。不得把提示词称为自动化、把计划称为写回。

## 3. 确定学科文件夹

根据当前材料、用户明确用语、关联目标和已有路径推断学科。学科专属状态放在 `subjects/<subject>/`。只有歧义会导致错误写入时才问一个简短问题；用户纠正始终覆盖自动推断。

`learning-policy.md`、`learner-profile.md` 与跨学科 `dashboard.md` 留在仓库根目录。不要预先创建空学科文件夹。

## 4. 读取最小状态

开始时只读：

1. 本文件；
2. `AGENTS.zh-CN.md`；
3. 目标仓库 `learning-policy.md`，存在时；
4. 根目录 `dashboard.md` 与当前学科入口；
5. `subjects/<subject>/` 内的活跃目标与当前输入真正相关的最少文件。

## 5. 识别意图

按最小操作路由：

- **setup：**首次搭建或迁移；
- **organize：**笔记、错题、老师反馈、平台结果、去重和状态同步；
- **question：**诊断、练习、变式、迁移、复习或给老师的问题包；
- **session：**用户明确希望现在由 AI 教学；
- **review：**已有题目需要作答、评分、调度或写回；
- **source / model：**来源记录或可复用理解；
- **maintenance：**陈旧、矛盾、重复、损坏或撤销。

支持 Skills 时通过 `skills/repo-as-review-os/SKILL.md` 路由。不要把所有学习请求默认送进 session。

## 6. 执行自动整理与写回

```text
捕获输入
→ 标记渠道、来源与隐私
→ 确定 subjects/<subject>/
→ 关联目标、缺口和已有状态
→ 去重
→ 整理、出题、交接、验证或只记录
→ 按 learning-policy 写回
→ 刷新 dashboard
→ 返回回执
```

默认 `safe-auto`：安全、低风险、可逆的普通学习状态直接更新并汇报；删除、公开、敏感信息、大规模重构和政策变化先询问。

## 7. 正确处理外部学习

老师、课堂、补习、纸笔和其他平台都是一等来源。

- “准备问老师” → 生成 `subjects/<subject>/handoffs/` 问题包并标记等待反馈；
- “老师已经解决” → 更新处理状态、同步反馈、清除过时任务；
- “这是我的笔记” → 记录来源、提炼必要模型；
- “不用再测” → 尊重选择，不安排验证；
- “撤销刚才” → 在安全可行时撤销最近一次原子学习更新。

问题已由外部解决不等于独立掌握，但也不应被 AI 强制重讲。

## 8. 正确出题

题目必须连接目标、来源或模型、知识缺口和近期表现。说明题目目的、难度、时间、答案或评分标准，并避免用刚示范过的原题验证。
