# 从这里开始：Agent 接手入口

GitLearnOS 是轻量教学套件，不是独立应用。使用本文件即可开始，不要先把整个模板仓库塞进上下文。

## 1. 识别两个仓库

```text
模板仓库：Guojiz/Repo-as-Review-OS
目标学习仓库：<用户仓库或未知>
```

绝不能把个人学习数据写进模板仓库。如果目标仓库未知，而且任务需要写回，只询问目标仓库这一项。

## 2. 检测真实能力

行动前在内部确认：

```text
当前运行环境：
模板读取权限：
目标仓读取权限：
目标仓写入权限：
已连接来源权限：
是否支持 Skill：
确实需要用户手动完成的步骤：
```

安全时应通过现有工具直接测试能力。当前环境自己能完成的事情，不要再让学习者手工操作。

## 3. 只读最小状态

开始时只读：

1. 本文件；
2. `AGENTS.zh-CN.md`；
3. 目标仓库的 `dashboard.md`；
4. 当前活跃目标；
5. `learner-profile.md` 与本次任务真正相关的来源、模型、缺口和复习文件。

不要在开始前通读所有模板文档。只有当前任务需要某项规则时，才打开更深层资料。

## 4. 选择一种模式

- **setup：**目标学习状态尚未建立；
- **session：**学习者想学习、继续、练习或接受讲解；
- **source：**需要诚实记录材料；
- **model：**需要提取可复用理解；
- **review：**需要生成或评分到期复习；
- **maintenance：**状态陈旧、矛盾或损坏。

如果支持 Skills，通过 `skills/repo-as-review-os/SKILL.md` 路由。

## 5. 运行学习内核

实际学习会话遵循：

```text
选择一个可观察目标
→ 先让学习者尝试或回忆
→ 诊断回答
→ 给出最小但有用的提示或讲解
→ 适合时做迁移检验
→ 证据评分 0–3
→ 只写回长期有价值且有证据的变化
→ 确定下一步
```

AI 讲解过，不等于学习者已掌握。学习者没有证据地说“懂了”，也不等于已掌握。必须诚实保留不确定性。

## 6. 明确收尾

汇报：

```text
改动文件：
- 路径：原因

学习证据：
- 实际观察结果与评分；未测试则写“未评估”

仍然需要：
- 缺少的来源、学习者尝试或决定

下一步：
- 一个具体动作
```
